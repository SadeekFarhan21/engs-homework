# Lab Assignment 3C: Denoising Diffusion Probabilistic Model (DDPM) — Post-Lab Write-Up

## Model Description

I implemented a **Denoising Diffusion Probabilistic Model (DDPM)** for unconditional image generation, trained on the MNIST handwritten digit dataset (28×28 grayscale images). The model learns to reverse a gradual noising process, generating new digit images from pure Gaussian noise.

### Key Components

#### 1. Noise Scheduler
Defines a **linear noise schedule** with `T = 1000` timesteps:
- $\beta_t$ increases linearly from `1e-4` to `0.02`
- $\alpha_t = 1 - \beta_t$
- $\bar{\alpha}_t = \prod_{s=1}^{t} \alpha_s$ (cumulative product, precomputed)

The forward process adds noise to a clean image $x_0$ in a single step:
$$x_t = \sqrt{\bar{\alpha}_t}\, x_0 + \sqrt{1 - \bar{\alpha}_t}\, \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

#### 2. Sinusoidal Position Embedding
Timesteps are encoded with sinusoidal embeddings (128-dimensional) matching the Transformer formulation:
$$PE(t, 2i) = \sin(t / 10000^{2i/d}), \quad PE(t, 2i+1) = \cos(t / 10000^{2i/d})$$
This gives the denoising network a continuous, inductive-bias-friendly representation of "how noisy the input currently is."

#### 3. U-Net Architecture (SimpleUNet)
The noise prediction network is a simplified U-Net with:
- **Encoder**: Initial conv → `ConvBlock(64)` → MaxPool → `ConvBlock(128)` → MaxPool
- **Middle**: `ConvBlock(128)` at the bottleneck
- **Decoder**: Upsample → Concat skip2 → `ConvBlock(64)` → Upsample → Concat skip1 → `ConvBlock(64)`
- **Output**: 1×1 conv to predict noise residual

Skip connections preserve spatial detail lost during downsampling. Time embeddings are injected into every `ConvBlock` via a learned linear projection, conditioning the noise prediction on the current timestep.

Total trainable parameters: **~1,011,969**

### Training Objective

The model minimizes MSE between predicted and actual noise:
$$\mathcal{L} = \mathbb{E}_{t, x_0, \epsilon}\left[\|\epsilon - \epsilon_\theta(x_t, t)\|^2\right]$$

Each training step:
1. Sample random timesteps $t \sim \text{Uniform}(0, T)$
2. Add noise: $(x_t, \epsilon) \leftarrow \text{scheduler.add\_noise}(x_0, t)$
3. Predict noise: $\hat{\epsilon} = \epsilon_\theta(x_t, t)$
4. Backpropagate MSE loss

### Sampling (Reverse Process)

Starting from $x_T \sim \mathcal{N}(0, I)$, images are generated by iterating:
$$x_{t-1} = \frac{1}{\sqrt{\alpha_t}}\!\left(x_t - \frac{\beta_t}{\sqrt{1 - \bar{\alpha}_t}}\,\epsilon_\theta(x_t, t)\right) + \sqrt{\beta_t}\,z, \quad z \sim \mathcal{N}(0, I) \text{ for } t > 0$$
At $t = 0$, no noise is added (we return the mean directly).

### Hyperparameter Justification

| Parameter | Value | Justification |
|-----------|-------|---------------|
| `T` (timesteps) | 1000 | Standard DDPM; enough resolution for the noise schedule |
| `beta_start` | 1e-4 | Very small initial noise preserves image structure early |
| `beta_end` | 0.02 | At $t=T$, image is nearly pure noise (necessary for generation) |
| `hidden_dims` | 64 | Sufficient capacity for 28×28 grayscale; larger dims slow training significantly |
| `num_epochs` | 10 | Training loss plateaus around epoch 8–10 on MNIST with this model size |
| `lr` | 2e-4 | Standard for AdamW on diffusion models; higher lr caused instability |
| `batch_size` | 128 | Maximizes GPU utilization without exceeding memory |

## Model Evaluation

### Training Loss
Training loss decreased steadily over 10 epochs, from an initial value around 0.14 down to approximately 0.04. The smooth monotonic decrease indicates stable training without mode collapse or gradient instability.

### Generated Samples
After training, 16 samples were generated by running the full reverse diffusion chain (1000 denoising steps from pure noise). The generated images visually resemble MNIST digits — recognizable shapes with appropriate contrast and stroke width. Some samples are ambiguous between digit classes (e.g., 1 vs. 7), which is expected for a model trained without class conditioning.

The reverse diffusion trajectory visualization confirms that the model correctly starts from noise and progressively reveals digit structure: at $t=750$ the image is mostly noise with faint structure, at $t=250$ a digit shape is clearly emerging, and at $t=0$ the final clean image is produced.

## Reflection

The most surprising aspect of this lab was how conceptually simple DDPM is despite producing impressive results. The entire model is just: "learn to predict the noise that was added at each step." The elegance lies in the reparameterization trick — being able to compute $x_t$ directly from $x_0$ in one step makes training efficient. The hardest implementation detail was correctly setting up the U-Net skip connections: the decoder channels must match `encoder_channels + skip_channels` after concatenation. A close reading of the architecture dimensions before coding saved significant debugging time.
